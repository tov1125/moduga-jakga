## 7. 교훈 및 개선 사항 (Lessons Learned)

### 7.1 What Went Well (성공 요인)

#### ✅ Excellent Points

1. **Early Service Integration**
   - 외부 API (OpenAI, CLOVA, Supabase)를 Sprint 2에 조기 통합
   - 실제 데이터로 검증 가능 → 설계 오류 조기 발견 가능
   - **Lesson**: MVP는 조기 외부 시스템 통합이 중요

2. **Pydantic Strict Type System**
   - 모든 스키마에 `StrictStr`, `StrictInt` 적용
   - 런타임 타입 오류 전무 → 신뢰도 높은 API
   - **Lesson**: 엄격한 타입이 개발 속도 향상 (버그 수정 시간 감소)

3. **Voice-First Architecture**
   - 초기부터 STT/TTS를 UI의 핵심으로 설계
   - 모든 컴포넌트가 키보드/음성 탐색 지원
   - **Lesson**: 접근성을 후처리 아닌 설계 단계에서 완성

4. **Clear Separation of Concerns**
   - 9개 서비스의 단일 책임 원칙 준수
   - 각 서비스 <500줄, 의존성 최소화
   - **Lesson**: 마이크로서비스 패턴이 테스트/유지보수 용이

5. **Comprehensive Error Handling**
   - 모든 엔드포인트에서 400/401/403/404/500 처리
   - 사용자 친화적 에러 메시지 + 스크린 리더 안내
   - **Lesson**: 에러 처리는 UX의 중요한 부분

6. **Documentation-First Approach**
   - 계획(Plan) → 설계(Design) → 구현(Do) 순서 준수
   - 각 단계별 문서 작성으로 방향성 명확화
   - **Lesson**: PDCA 문서화가 혼선 방지 + 학습 자산화
