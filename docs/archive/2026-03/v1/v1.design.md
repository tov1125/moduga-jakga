## 1. Overview

### 1.1 Design Goals

- Voice-First: 모든 주요 조작을 음성으로 수행 가능
- 접근성: WCAG 2.1 AA, 스크린 리더 호환, 키보드 전용 탐색
- 타입 안전: Pydantic Strict + TypeScript strict
- 실시간: SSE 글 생성 스트리밍, WebSocket STT

### 1.2 Design Principles

- 음성 우선 (Voice-First): 시각 의존도 최소화
- 최소 단계 (Minimal Steps): 목표까지 최소 인터랙션
- 지속적 피드백 (Continuous Feedback): ARIA live 영역으로 상태 안내
- 엄격한 타입 안전성: BE Pydantic Strict + FE TypeScript strict

## 2. Architecture

### 2.1 System Architecture

```
사용자 (음성 입력)
    │
    ▼
┌──────────────────┐    JWT     ┌──────────────────┐
│  Next.js 15 FE   │◀─────────▶│  FastAPI BE      │
│  (App Router)    │  REST/SSE  │  (Pydantic v2)   │
│  Port: 3000      │            │  Port: 8000      │
└──────────────────┘            └────────┬─────────┘
                                         │
                    ┌────────────────────┤────────────────────┐
                    ▼                    ▼                    ▼
            ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
            │  Supabase    │    │  OpenAI      │    │  CLOVA       │
            │  (DB+Auth)   │    │  (GPT-4o)    │    │  (STT/TTS)   │
            └──────────────┘    └──────────────┘    └──────────────┘
```

### 2.2 인증 흐름

```
[Frontend]                    [Backend]                   [Supabase]
    │                            │                            │
    │── POST /auth/login ───────▶│                            │
    │                            │── sign_in_with_password ──▶│
    │                            │◀── session ────────────────│
    │                            │── create JWT ──────────────│
    │◀── { user, token } ───────│                            │
    │                            │                            │
    │── localStorage.setItem ───│                            │
    │   ("access_token", token) │                            │
    │                            │                            │
    │── GET /auth/me ───────────▶│── verify JWT ─────────────│
    │   (Authorization: Bearer)  │── query profiles ─────────▶│
    │◀── { user } ──────────────│◀── profile data ───────────│
```

### 2.3 Data Flow

```
음성 입력 → STT(CLOVA) → 텍스트 → AI 글쓰기(GPT-4o) → 편집/교열 → 출판(DOCX/PDF/EPUB)
     │                                                        │
     └── TTS 피드백(CLOVA Voice) ◀────────────────────────────┘
```

```json

  "success": true,
  "data": { ... }
}
```

**페이지네이션 응답:**
```json
{
  "success": true,
  "data": [...],
  "total": 100,
  "page": 1,
  "page_size": 20,
  "total_pages": 5
}
```

**에러 응답:**
```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "입력값이 올바르지 않습니다.",
    "details": { "field": ["error message"] }
```

## 8. Error Handling

### 8.1 Backend 에러 코드

| Code | Message | Cause |
|------|---------|-------|
| 400 | 입력값이 올바르지 않습니다 | Pydantic 검증 실패 |
| 401 | 인증이 필요합니다 | JWT 만료/무효 |
| 403 | 접근 권한이 없습니다 | RLS 위반 |
| 404 | 리소스를 찾을 수 없습니다 | 도서/챕터 미존재 |
| 500 | 서버 내부 오류 | 외부 서비스 장애 |

### 8.2 Frontend 에러 처리

- `apiFetch()`: 401 → localStorage 토큰 삭제 → `/login` 리다이렉트
- API 호출 실패 → ARIA live 영역으로 에러 메시지 안내
- 네트워크 오류 → 재시도 안내 표시

2. 도서 선택 → 글쓰기 워크스페이스 (챕터 + 에디터)
3. 편집 페이지 (4단계 탭)
4. 설정 페이지 (프로필 + 음성)
5. 로그아웃 → 홈 리다이렉트
6. 회원가입 흐름
