## 1. Overview

### 1.1 Purpose

시각장애인 사용자가 음성으로 대화하면 AI가 글을 작성하고, 편집/교열을 거쳐 책(DOCX/PDF/EPUB)으로 출판할 수 있는 웹 애플리케이션의 v1 MVP를 구축한다.

### 1.2 Background

- "말하다 → 글이 되다 → 책이 되다 → 작가가 되다" 핵심 가치 실현
- Sprint 1: Backend 뼈대 + Pydantic 스키마 + Frontend 접근성 기반 UI
- Sprint 2: 외부 서비스 연동 (OpenAI, CLOVA STT/TTS, Supabase)
- Sprint 3: FE↔BE 통합 + 편집 서비스 PDCA 완료

### 1.3 Related Documents

- CLAUDE.md: 프로젝트 설계 원칙 및 코딩 컨벤션
- agent.md: 에이전트 체계 (A0~A18)

## 2. Scope

### 2.1 In Scope

- [x] 사용자 인증 (회원가입/로그인/로그아웃/설정)
- [x] 도서 CRUD (생성/조회/수정/삭제)
- [x] 챕터 CRUD (생성/조회/수정/삭제, 순서 관리)
- [x] AI 글쓰기 (SSE 스트리밍 생성/재작성/구조 제안)
- [x] 4단계 편집/교열 (구조편집/내용편집/교정/최종검토)
- [x] TTS 음성 합성 (CLOVA Voice, 8명 화자)
- [x] STT 음성 인식 (CLOVA Speech, WebSocket 실시간)
- [x] 책 디자인 (AI 표지 생성, Typst 내지 레이아웃)
- [x] 출판/내보내기 (DOCX/PDF/EPUB)
- [x] 웹 접근성 (WCAG 2.1 AA, 키보드 내비게이션, ARIA)
- [x] FE↔BE JWT 기반 인증 통합

### 2.2 Out of Scope

- ISBN 발급 프로세스
- 전자책 플랫폼 연동 (리디북스, 밀리의서재 등)
- POD(주문형 인쇄) 연동
- 다국어 지원
- 실사용자(시각장애인) 현장 테스트

## 5. Risks and Mitigation

## 6. Architecture Considerations

### 6.1 Project Level Selection

| Level | Characteristics | Selected |
|-------|-----------------|:--------:|
| Starter | Simple structure | |
| Dynamic | Feature-based modules, BaaS | |
| **Enterprise** | Strict layer separation, DI, microservices | **O** |

### 6.2 Key Architectural Decisions

| Decision | Options | Selected | Rationale |
|----------|---------|----------|-----------|
| Frontend | Next.js App Router | Next.js 15 | SSR/CSR 혼합, 접근성 우수 |
| Backend | FastAPI + Pydantic | FastAPI | 비동기, 타입 안전, 자동 문서화 |
| DB | Supabase (PostgreSQL) | Supabase | Auth 통합, RLS, 실시간 |
| AI | OpenAI GPT-4o | GPT-4o | 한국어 품질, SSE 스트리밍 |
| STT | CLOVA Speech | CLOVA | 한국어 인식률 최고 |
| TTS | CLOVA Voice | CLOVA | 한국어 8명 화자, 자연스러움 |
| 조판 | Typst | Typst | 경량, PDF 출력, 한글 지원 |
| 인증 | Supabase Auth + JWT | JWT | BE-FE 분리 인증 |
