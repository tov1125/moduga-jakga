# VoiceOver 접근성 수동 테스트 체크리스트

> 대상 스크린 리더: macOS VoiceOver (Cmd+F5)
> 대상 브라우저: Safari (VoiceOver 최적 호환)
> 테스트 기준: WCAG 2.1 AA

---

## 1. 홈페이지 (`/`)

- [ ] 페이지 제목이 "모두가 작가 - AI 글쓰기 플랫폼"으로 읽힘
- [ ] 건너뛰기 링크("본문으로 이동")가 첫 번째 탭으로 포커스됨
- [ ] 로고 링크에 "모두가 작가 - 홈으로 이동" aria-label 읽힘
- [ ] 네비게이션 랜드마크(nav role)가 인식됨
- [ ] CTA 버튼("시작하기", "로그인")이 정상 읽힘
- [ ] 테마 토글 버튼 aria-label이 현재 동작을 설명함
- [ ] 탭 순서가 논리적 (로고 → 네비 → 본문 → 푸터)

## 2. 로그인 (`/login`)

- [ ] 페이지 제목이 "로그인 | 모두가 작가"로 읽힘
- [ ] 폼 aria-label "로그인 양식"이 인식됨
- [ ] 이메일 입력란: label 연결, placeholder, aria-required 읽힘
- [ ] 비밀번호 입력란: label 연결, aria-required 읽힘
- [ ] 에러 발생 시 role="alert"로 즉시 에러 메시지 읽힘
- [ ] 로그인 버튼 상태(로딩 중) 변화가 안내됨
- [ ] "회원가입" 링크가 읽히고 포커스 가능

## 3. 회원가입 (`/signup`)

- [ ] 폼 aria-label "회원가입 양식"이 인식됨
- [ ] 모든 필수 필드에 aria-required="true" 읽힘
- [ ] 이메일 힌트 텍스트(aria-describedby)가 읽힘
- [ ] 비밀번호 힌트 "8자 이상"이 읽힘
- [ ] 시각 장애 유형 select의 옵션이 모두 탐색 가능
- [ ] 이용약관 체크박스 + 링크가 읽힘 ("이용약관에 동의합니다")
- [ ] 개인정보처리방침 체크박스 + 링크가 읽힘
- [ ] AI 저작권 동의 체크박스가 읽힘
- [ ] 모든 체크박스 미체크 시 가입 버튼 비활성화 상태 읽힘
- [ ] 필드 유효성 에러가 role="alert"로 읽힘

## 4. 대시보드 (`/dashboard`)

- [ ] 사용자 이름이 헤더에서 읽힘 ("{이름}님으로 로그인됨")
- [ ] 책 목록이 리스트로 탐색 가능
- [ ] "새 작품 만들기" 버튼이 읽힘
- [ ] 각 작품 카드에 제목, 장르, 상태 정보가 읽힘
- [ ] 빈 상태 메시지가 읽힘

## 5. 글쓰기 (`/write/[bookId]`)

- [ ] 페이지 제목에 책 이름 포함
- [ ] 챕터 선택 드롭다운이 탐색 가능
- [ ] 텍스트 편집 영역에 label 연결
- [ ] STT 녹음 버튼 aria-label "녹음 시작/중지"가 읽힘
- [ ] STT 상태 변화 안내 (aria-live): "녹음을 시작합니다" / "녹음을 중지합니다"
- [ ] 중간 인식 결과(interim)가 aria-live="polite"로 읽힘
- [ ] AI 글쓰기 버튼과 로딩 상태가 안내됨

## 6. 편집 (`/write/[bookId]/edit`)

- [ ] 편집 단계 탭(구조/내용/교정/교열)이 탭 패널로 탐색됨
- [ ] "분석 실행" 버튼과 로딩 상태 안내
- [ ] 제안 목록이 읽히고 개별 탐색 가능
- [ ] 수락/거부 버튼에 적절한 aria-label
- [ ] "모두 수락" 버튼이 읽힘
- [ ] 되돌리기(Undo) 버튼의 비활성화 상태 읽힘
- [ ] 다시 실행(Redo) 버튼의 비활성화 상태 읽힘
- [ ] Ctrl+Z / Ctrl+Shift+Z 단축키 동작 확인
- [ ] 수정 적용 시 "수정이 적용되었습니다" aria-live 안내
- [ ] 저장 중 상태가 role="status"로 안내됨
- [ ] 품질 보고서 섹션 읽힘

## 7. 출판 (`/write/[bookId]/publish`)

- [ ] 표지 디자인 선택 영역이 라디오 그룹으로 탐색됨
- [ ] 템플릿 썸네일에 alt 텍스트
- [ ] 출력 형식(DOCX/PDF/EPUB) 선택이 읽힘
- [ ] "출판하기" 버튼과 로딩 상태
- [ ] 완료 시 다운로드 링크가 읽힘
- [ ] 에러 시 role="alert"로 안내

## 8. 설정 (`/settings`)

- [ ] 이름 변경 필드와 label 연결
- [ ] 시각 장애 유형 변경 가능
- [ ] 저장 성공/실패 안내
- [ ] 로그아웃 버튼이 읽힘

## 9. 법률 페이지 (`/terms`, `/privacy`)

- [ ] 제목이 h1으로 읽힘
- [ ] 섹션별 h2 헤딩으로 탐색 가능
- [ ] 목록 항목이 ul/li로 구조화됨
- [ ] 다크모드에서 텍스트 대비 충분 (4.5:1 이상)

---

## 공통 체크 항목

- [ ] 모든 포커스 가능한 요소에 `focus-visible:ring` 표시 확인
- [ ] 모든 버튼의 최소 터치 영역 44x44px (`min-h-touch`)
- [ ] 색상만으로 정보를 전달하지 않음 (아이콘/텍스트 보조)
- [ ] 모든 이미지에 대체 텍스트
- [ ] 다크모드 전환 시 모든 텍스트 대비 4.5:1 이상 유지
- [ ] 텍스트 200% 확대 시 레이아웃 깨지지 않음
